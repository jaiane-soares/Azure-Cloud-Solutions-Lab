
### 🪤 Tabela de pegadinhas: AZ-900

| O que a prova diz  | A realidade | Dica  |
| :--- | :--- | :--- |
| Escalabilidade vertical aumenta o número de máquinas. | Escalabilidade vertical aumenta o "poder" (CPU/RAM) da máquina atual. | Vertical = Up/Down (Upgrade). Horizontal = Out/In (Mais máquinas). |
| Região e zona de disponibilidade são sinônimos. | Região é uma área geográfica; zona de disponibilidade é um datacenter isolado dentro da região. | Uma região pode ter várias zonas. Zonas protegem contra falhas de datacenter. |
| Resource groups podem conter outros grupos de recursos. | Resource groups não podem ser aninhados (um dentro do outro). | São recipientes de nível único para organizar seus "brinquedos". |
| Azure Advisor monitora a saúde dos serviços em tempo real. | Azure Advisor dá recomendações de otimização (custo, segurança, performance). | Para monitorar métricas de CPU/Rede em tempo real, use o **Azure Monitor**. |
| Azure Policy serve para dar permissão a um usuário. | Azure Policy foca na conformidade dos recursos (ex: impedir criação de VM cara). | Permissão de usuário (quem faz o quê) é com o **Azure RBAC**. |
| Nuvem pública é inerentemente menos segura que a privada. | A segurança depende da configuração e do modelo de responsabilidade compartilhada. | No Azure, você herda a segurança física da Microsoft, que é gigante. |
| Gerenciamento de custos (Cost Management) impede gastos excedentes automaticamente. | O Cost Management gera alertas e relatórios, mas não "trava" o serviço sozinho. | Você precisa configurar orçamentos (Budgets) para ser avisada antes do susto. |
| Transferência de dados entre regiões é sempre gratuita. | Dados saindo de uma região para outra (egresso) geralmente são cobrados. | Entrada (ingresso) é grátis. Saída custa dinheiro. Cuidado com o orçamento! |
